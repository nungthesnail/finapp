# TASKS: AI-финансовый консультант

Источник требований: `SPEC.md`  
Формат: этапы -> задачи -> критерии готовности

## Этап 0. Инициализация проекта

- [ ] Подготовить монорепо-структуру (`apps/web`, `apps/api`, `docs`, `infra`, `scripts`).
Критерии: каталоги созданы, базовые README для web/api добавлены.

- [ ] Настроить окружение разработки (Docker Compose: web, api, mysql, nginx).
Критерии: проект поднимается одной командой, API и web доступны локально.

- [ ] Настроить базовые CI-пайплайны (web lint/test, api test, e2e smoke).
Критерии: CI запускается на PR, падает при ошибках тестов/линта.

## Этап 1. Foundation (Auth + базовые финансы)

- [ ] Реализовать регистрацию, логин, logout, профиль пользователя.
Критерии: API endpoints работают, есть feature-тесты.

- [ ] Ввести роли `USER` и `ADMIN`, ограничить доступ к admin API.
Критерии: RBAC-политики и тесты на доступ/запрет.

- [ ] Реализовать CRUD счетов.
Критерии: создание/редактирование/удаление/просмотр, валидация и тесты.

- [ ] Реализовать CRUD доходов/расходов с фильтрами.
Критерии: фильтры по периоду/типу/категории/счету, тесты покрывают основные сценарии.

- [ ] Реализовать базовый UI: auth, dashboard, счета, операции.
Критерии: пользователь проходит полный путь без ручных API-вызовов.

## Этап 2. Planning (периодика + бюджетные планы + аналитика)

- [ ] Реализовать периодические доходы/расходы.
Критерии: есть модель периодичности, фоновая генерация операций, тесты.

- [ ] Реализовать бюджетные планы (период, категории, целевые покупки, цель).
Критерии: CRUD планов и вложенных сущностей, валидация связей со счетами.

- [ ] Реализовать аналитику доходов/расходов (summary, timeseries, categories).
Критерии: API-агрегаты, корректные расчеты, unit/feature-тесты.

- [ ] Построить UI-таблицы и графики по аналитике.
Критерии: фильтры синхронизированы с API, данные отображаются корректно на mobile/desktop.

## Этап 3. AI Core

- [ ] Реализовать чат с AI и streaming response.
Критерии: стабильный поток ответа в UI, сохранение диалогов и сообщений в БД.

- [ ] Реализовать реестр AI-моделей и тарифов токенов.
Критерии: модели и ценовые параметры хранятся в БД и доступны в настройках.

- [ ] Реализовать tool/function layer для безопасных действий агента.
Критерии: агент не имеет прямого доступа к произвольным данным, только разрешенные инструменты.

- [ ] Реализовать AI-функции: прогноз, создание/редактирование плана, контроль плана.
Критерии: каждый сценарий доступен через чат и подтверждается тестовыми кейсами.

- [ ] Реализовать учет потребления токенов и стоимости.
Критерии: логируются input/output/cache tokens, стоимость и связь с пользователем.

## Этап 4. Billing (подписки, платежи, кредиты)

- [ ] Реализовать сущности тарифов и подписок.
Критерии: тарифы управляются из админки, подписка привязана к пользователю.

- [ ] Интегрировать YooMoney checkout и webhook.
Критерии: успешная оплата продлевает подписку, webhook идемпотентен.

- [ ] Реализовать credit ledger и списание за AI-запросы.
Критерии: все начисления/списания идут через ledger, операции транзакционны.

- [ ] Реализовать политику сгорания кредитов.
Критерии: ежедневная job корректно списывает просроченные кредиты.

- [ ] Добавить экран подписки/потребления для пользователя.
Критерии: видны текущий баланс, история списаний, смена тарифа.

## Этап 5. Notifications + Support + Admin

- [ ] Реализовать in-app уведомления и статусы прочтения.
Критерии: список уведомлений доступен, mark-as-read работает.

- [ ] Реализовать расписание AI-контроля планов и генерацию уведомлений.
Критерии: плановое задание запускается по расписанию и создает уведомления.

- [ ] Реализовать чат техподдержки user-admin.
Критерии: двусторонняя переписка с хранением истории.

- [ ] Реализовать админ-панель: пользователи, тарифы, подписки, платежи, usage.
Критерии: ключевые операции администратора выполняются из UI.

- [ ] Добавить аудит действий администратора.
Критерии: действия пишутся в audit log и доступны для проверки.

## Этап 6. Legal + Hardening

- [ ] Подготовить landing, privacy policy, help pages.
Критерии: страницы доступны публично и связаны с основными сценариями.

- [ ] Провести security hardening (валидация, rate limit, защита секретов, логирование).
Критерии: закрыты базовые риски OWASP для API/UI.

- [ ] Добавить политику хранения/удаления персональных данных (152-ФЗ).
Критерии: документирован процесс, реализованы необходимые механизмы в системе.

- [ ] Настроить observability (логи, метрики API, AI-cost метрики, алерты).
Критерии: ключевые метрики и ошибки видны в мониторинге.

- [ ] Провести стабилизацию релиза (регресс, perf smoke, bugfix).
Критерии: MVP-сценарии проходят без блокирующих дефектов.

## Поперечные задачи (выполняются на всех этапах)

- [ ] Поддерживать документацию (`SPEC.md`, `docs/api/openapi.yaml`, ADR).
Критерии: изменения контрактов всегда отражены в документах.

- [ ] Поддерживать тестовое покрытие критичных сценариев.
Критерии: новые фичи без тестов не считаются завершенными.

- [ ] Контролировать качество UX на мобильных устройствах.
Критерии: все основные user-flow корректно работают на мобильных брейкпоинтах.
